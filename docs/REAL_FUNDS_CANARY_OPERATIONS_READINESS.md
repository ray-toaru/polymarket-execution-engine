# Real Funds Canary Operations Readiness

This document defines local auditable prerequisites only. It does not approve
production deployment, live submit, live cancel, or an armed real-funds canary.

Required references before any future armed canary:

- reviewed release decision JSON bound to the current artifact SHA-256 and
  evidence manifest SHA-256; the template must be default no-go;
- external reference package bound to the current artifact SHA-256, evidence
  manifest SHA-256, and GitHub evidence run IDs;
- external secret provider reference, rotation evidence reference, and
  break-glass review reference;
- external alert routing reference covering runtime worker health, remote
  unknown count, SDK error rate, audit export failure, and idempotency conflict
  rate;
- rollback runbook covering read-only, sign-only, and cancel-only fallback;
- canary retry policy that waits for safe market conditions instead of
  weakening market-safety thresholds without review.

The local review package generated by
`validation/prepare_real_funds_canary_review.py` is review material only. It
must include `external-references.json` and `release-decision.json`, and report
`live_submit_allowed=false`, `live_cancel_allowed=false`,
`real_funds_canary_authorized=false`, `remote_side_effects=false`, and
`secrets_included=false`.
